# 脚手架操作约定

## 目的

`README.md` 为本项目**唯一入口文档**。在已有仓库内再次执行 Vite / 同类脚手架时，若误选覆盖选项，会替换 `README.md` 为模板默认内容。

## 约定

1. **禁止**在提示「目录非空」时选择「删除已有文件并继续」。
2. **应选**「忽略已有文件并继续」，或先在**空目录**初始化，再将代码迁入本仓库。
3. 执行任何会写入根目录的脚手架前，先用 Git 提交或备份 `README.md`；若已被覆盖，从版本库恢复该文件。
4. 新增子应用或模块时，优先在**子目录**内初始化，避免在仓库根目录重复跑 `create vite`。

## 恢复 `README.md`

若已被覆盖：使用 Git 从历史版本还原 `README.md`，或从备份恢复后再提交。
