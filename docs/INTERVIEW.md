# Vue 3 企业级电商平台前端项目介绍

## 一、项目概述

该项目是一个面向企业级业务场景的电商前端平台实践，采用 Vue 3 技术栈完成从商品浏览、交易下单到营销活动与用户成长体系的完整闭环。  
项目定位为“**全栈协作友好的前端主导型工程**”：在本地 Mock 数据环境下可独立演示复杂业务流程，同时具备与真实后端 API 平滑对接能力，适合作为中大型前端项目架构能力与工程能力的展示样例。

核心价值：

- 以真实业务链路驱动前端架构设计与模块拆分
- 同时覆盖 PC 与移动端，验证多端一致性与差异化体验设计能力
- 体现“功能实现 + 工程规范 + 性能优化 + 可演进性”的综合能力

## 三、架构设计亮点

### 3.1 Feature-Sliced Design（FSD）分层架构

项目采用 FSD 思想组织 `src` 目录，按业务职责拆分为 `app/pages/widgets/features/entities/stores/shared`，实现“高内聚、低耦合”的结构目标：

- `app` 负责应用装配、路由守卫与入口初始化
- `pages` 负责页面级编排
- `features` 承载可复用业务能力（筛选、签到、红包、优惠券等）
- `entities` 维护领域模型与类型边界
- `stores` 集中管理跨页面状态
- `shared` 提供 API、通用组件、常量和工具

### 3.2 PC / 移动端双布局自适应

在 `App.vue` 中通过 `useMediaQuery('(min-width: 1024px)')` 动态切换 `PcLayout` 与 `MobileLayout`，形成“同一路由、多端容器”的架构模式：

- 业务逻辑复用：同一页面数据逻辑在两端复用
- 交互差异化：PC 偏信息密度与操作效率，移动端偏流程连贯与触控体验
- 降低维护成本：避免双工程并行维护

### 3.3 状态管理分层（Vue Query + Pinia）

- **服务端状态（Server State）**：由 Vue Query 管理查询请求、加载状态、缓存与重取策略
- **客户端状态（Client State）**：由 Pinia 管理用户、购物车、订单、地址、优惠券等业务状态

该分层避免“所有状态都堆进 Store”导致的复杂度膨胀，提升状态边界清晰度。

## 五、技术难点与解决方案

### 5.1 优惠券门槛跨类目计算逻辑

- **问题**：若按“整单是否同类目”判断优惠券可用性，会导致跨类目订单无法正确参与类目券抵扣。
- **分析**：真实业务通常按“命中类目商品小计”判断券门槛，而非整单强约束。
- **方案**：在 `useCoupon` 中改为按“匹配类目商品 subtotal”计算可用性。
- **效果**：提升券规则准确性，避免“券可见但不可用”体验问题。

### 5.2 秒杀商品日期绑定与折扣动态生成

- **问题**：秒杀商品若纯随机，每次刷新结果不一致，影响活动可信度。
- **分析**：需要“同一天稳定、次日刷新”的确定性随机策略。
- **方案**：使用“日期种子 + 伪随机”方式抽样商品并生成折扣。
- **效果**：同日结果稳定，可预测调试；跨日自动切换，具备运营节奏感。

### 5.3 积分发放与支付时机同步

- **问题**：积分若在“提交订单”即发放，会与实际支付行为脱节，造成业务漏洞。
- **分析**：积分属于支付后奖励，应与订单支付状态强绑定。
- **方案**：将发积分逻辑从 `createOrder` 挪至 `payOrder`，仅在待付款 -> 待发货流转时执行。
- **效果**：积分结算时机准确，状态一致性增强。

### 5.4 PC/移动端组件库混用的样式与交互治理

- **问题**：Vant 与 Element Plus 混用可能带来样式冲突、布局不一致与交互差异。
- **分析**：核心风险在于全局样式污染与组件职责边界不清。
- **方案**：通过布局容器层隔离端侧 UI，结合 Tailwind 原子类与 scoped 样式控制影响范围。
- **效果**：实现“一套业务，两端体验”，且可维护性良好。

### 5.5 Mock 数据与真实 API 的平滑切换

- **问题**：前后端并行开发时，接口不稳定会拖慢业务联调。
- **分析**：需要在请求层与数据层预留切换能力。
- **方案**：采用统一请求封装 + 本地 mock 模块，保持页面调用协议稳定。
- **效果**：开发期可独立推进复杂页面，实现后期低成本替换真实接口。

- 使用 ESLint + Prettier 实现“质量规则 + 风格规则”双重约束

## 七、性能优化实践

### 7.1 已落地优化

- 路由级懒加载：页面按需加载，减少首屏包体积
- Vue Query 缓存：减少重复请求与加载抖动
- 状态持久化：降低重复输入与回流成本，提升可用性
- 双端布局按需渲染：仅渲染当前端侧容器，降低无效渲染开销

### 7.2 适配与体验优化

- 移动端通过 `postcss-px-to-viewport` 做尺寸适配
- 关键页面处理底部安全区与 Tabbar 遮挡问题
- PC 端采用最大宽度与信息分区布局，提升信息可读性

> 说明：当前项目未引入虚拟滚动，列表性能主要通过分页、按需加载与缓存策略保障。
